# CI Fix Plan — 2026-04-12

> Author: Hangyuan (heartbeat audit)
> Status: Ready for implementation

## Problem Summary

Both `pr.yml` and `release.yml` will fail on any CI runner because:

1. **No `package-lock.json`** — project uses bun, only has `bun.lock`. `npm ci --frozen-lockfile` requires `package-lock.json`
2. **`npm install --package-lock-only` crashes** — npm arborist bug with monorepo workspace symlinks (TypeError: Cannot read properties of null)
3. **`--project integration` invalid** — no vitest workspace config exists; integration tests live in `tests/integration/` as a standalone sub-package
4. **`tests/infra` has own package.json** — needs separate install step

## Decision: Bun in CI

Since npm literally cannot resolve the workspace, and bun is the project's actual package manager:
- Use `oven-sh/setup-bun@v2` in CI
- Use `bun install --frozen-lockfile` (reads `bun.lock`)
- Keep `npx vitest run` for test execution (better-sqlite3 needs Node.js, not Bun runtime)

## Fix: pr.yml

### All jobs — replace setup + install steps:

```yaml
      - name: Set up Bun
        uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - name: Set up Node
        uses: actions/setup-node@v4
        with:
          node-version: '22'

      - name: Install dependencies
        run: bun install --frozen-lockfile
```

Note: Node is still needed for vitest/tsc execution. Bun is only the package manager.

### `integration` job — fix test command:

```yaml
      - name: Install integration test deps
        run: cd tests/integration && bun install --frozen-lockfile

      - name: Run integration tests
        run: cd tests/integration && npx vitest run --reporter=verbose
```

(Remove `--project integration` flag entirely)

### `infra-tests` job — fix install:

```yaml
      - name: Install infra test deps
        run: cd tests/infra && bun install

      - name: Run infra tests
        run: cd tests/infra && npx vitest run
```

### `lint-typecheck` job — add missing tsconfigs:

Check that `packages/protocol/tsconfig.json` and `packages/skill/tsconfig.json` exist. If `supernode/tsconfig.json` exists, add it too.

### `security` job — npm audit alternative:

`npm audit` won't work without `package-lock.json`. Options:
- (a) Remove security job for now (YAGNI for pre-release)
- (b) Use `bun audit` if available (check bun version)
- (c) Use `npx audit-ci` with a lockfile conversion step

Recommend (a) — remove until first public release.

## Fix: release.yml
