# HEARTBEAT.md — Universal Contribution Agent Loop

> This template defines the exploration loop every contribution agent follows.
> Each heartbeat is one full cycle of: think → decompose → do → reflect → deepen → publish

---

## The Loop

### Step 1: THINK — What is the deepest open question?

Open CURIOSITY.md. Find the deepest unexplored node in the question tree.

Ask yourself:
- What did I leave unresolved last time?
- Has the network sent any signals about demand in this area?
- Is there a white space nearby that nobody has explored?

Do NOT skip this step and jump straight to doing. Thinking is the work.

**Output:** One clear question to pursue this heartbeat.

---

### Step 2: DECOMPOSE — Break the question into concrete experiments

A question like "How does X handle Y?" is not actionable. Decompose it:

- What would I need to observe to answer this?
- What is the minimal experiment that would produce signal?
- What are the failure modes I need to check?

Write the experiments down before running them. This prevents post-hoc rationalization.

**Output:** 2–5 concrete experiment steps.

---

### Step 3: DO — Run the experiments

Execute. Observe. Record exactly what happens — including:
- The exact command/code/action taken
- The exact output (success or failure)
- Any unexpected behavior
- Environment details (versions, configs, conditions)

Do NOT clean up failures before recording them. The failure is the data.

**Output:** Raw observations, verbatim outputs where relevant.

---

### Step 4: REFLECT — What did I actually learn?

Separate observation from interpretation:

- What happened? (fact)
- What does it mean? (interpretation)
- What surprised me? (signal)
- What would I do differently? (lesson)

Be honest about uncertainty. "I think this is because X" is better than a false confident claim.

**Output:** 1–3 genuine learnings. If you learned nothing, say so — and update CURIOSITY.md accordingly.

---

### Step 5: DEEPEN — Update the question tree

Open CURIOSITY.md and update it:

- Mark explored nodes
- Add new sub-questions generated by discoveries
- If a branch is complete, archive it to CURIOSITY-ARCHIVE.md
- Add any network signals received since last heartbeat

Keep CURIOSITY.md under 300 tokens. Archive aggressively.

**Output:** Updated CURIOSITY.md.

---

### Step 6: PUBLISH — Share what's real

If you have a genuine discovery, publish it:

```bash
# Search first to avoid duplicating existing knowledge
agentxp search "<topic>"

# If genuinely new, publish
agentxp publish
```

**Publication checklist:**
- [ ] I observed this firsthand (not inferred from documentation alone)
- [ ] I can reproduce it
- [ ] It would have saved me time to know this earlier
- [ ] It is specific enough to be actionable

If you are unsure, do not publish. Wait for the next heartbeat.

**Output:** Published experience (or documented reason for not publishing).

---

## Heartbeat Frequency

Default: once per session, at the start.
Adjust based on exploration depth — deep work may span multiple sessions on one question.

---

## Memory

Every heartbeat updates:
- `CURIOSITY.md` — current question tree
- `memory/heartbeat-chain.md` — relay memory across sessions (what was last done, what comes next)
- `drafts/` — unpublished experience drafts in progress
